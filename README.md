# Шаблоны проектирования на Java
- Шаблон проектирования - повторяемая архитектурная конструкция, представляющая собой решение проблемы проектирования в рамках некоторой часто возникающей задачи.     
- В качестве примера представлены некоторые шаблоны и их возможные применения в рамках компьютерной игры.


* ## Порождающие шаблоны/Creational patterns - создание объектов
  - ### [Мультитон/Multiton](https://github.com/GarlicRoll/Software-design-pattern/tree/multiton) - конфигурации единичных объектов
  - ### [Абстрактная фабрика/Abstract factory](https://github.com/GarlicRoll/Software-design-pattern/tree/abstract_factory) - абстрактные группы классов
  - ### [Cтроитель/Builder](https://github.com/GarlicRoll/Software-design-pattern/tree/builder) - сложные объекты по частям

* ## Структурные шаблоны/Structural patterns - расположение объектов
  - ### [Мост/Bridge](https://github.com/GarlicRoll/Software-design-pattern/tree/bridge) - независимые группы классов
  - ### [Компоновщик/Composite](https://github.com/GarlicRoll/Software-design-pattern/tree/composite) - древовидная структура
  - ### [Декоратор/Decorator](https://github.com/GarlicRoll/Software-design-pattern/tree/decorator) - оборачивание классов

* ## Поведенческие шаблоны/Behavioral patterns - взаимодействие объектов
  - ### [Стратегия/Strategy](https://github.com/GarlicRoll/Software-design-pattern/tree/strategy) - разные стратегии
  - ### [Хранитель/Memento](https://github.com/GarlicRoll/Software-design-pattern/tree/memento) - сохранение
  - ### [Состояние/State](https://github.com/GarlicRoll/Software-design-pattern/tree/state) - состояния объекта
